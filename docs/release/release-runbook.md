# Release Runbook

1. Merge only with passing `CI Governance` workflow.
2. Ensure changeset entry exists for release-impacting package changes.
3. Validate release integrity:
   - `npm run check:release-readiness`
   - `npm run check:publish-ready`
4. Merge release PR generated by Changesets.
5. Confirm tag/publish workflow output for both packages.
6. If rollback is needed:
   - revert release commit on `main`
   - republish last known good version
   - document incident and compatibility impact

## Governance validation flow

- Runtime contract drift
- Local authority bypass prevention
- AOC boundaries and dependency direction
- Compatibility governance checks
- Lifecycle governance checks
